The Friendly Fedora Project

The "Friendy Fedora" Project is an on-going 5-year project being completed by UMass Lowell Electrical and Computer Engineering students, and sponsered by Red Hat Inc.
with a goal to create a more friendly Fedora distribution specifically customized for the UMass Lowell community and to promote the use of the Fedora Linux distribution.

Learn More

What is the Friendly Fedora Project?


The Friendly Fedora Project is an initiative undertaken by Red Hat Inc. to promote the use of the Fedora Linux distribution. The University of Massachusetts Lowell has undertaken their version of the project, which is to create a Fedora Linux distribution that is fully customized to the UMass Lowell community, benefitting students and faculty with popular applications, streamlined GUI features, and useful tools.

Linux is an opearting system that has many uses and is widely used for many different purposes across a wide range of disciplines. Fedora is becoming a more popular Linux distribution to use for many reasons. By creating a more friendly and customized system for UMass Lowell, students and faculty will have access to a powerful tool to better serve their needs in classes, projects, and many other use cases. It will also provide benefits to many by introducing them to Linux and useful open-source tools. Red Hat's "What is Linux" article is a great introduction to Linux.

This project is sponsered by Red Hat Inc. and is being completed by senior Electrical and Computer Engineering students for their Capstone project sequence.

Friendly Fedora Project Progress: Fall 2022 - Spring 2023


The Fall 2022-2023 Friendly Fedora Capstone team made substantial progress towards the project goal in
creating a Fedora distribution which is more "friendly" and geared towards the UMass Lowell community of students and faculty.

In this third-year of the project, we worked on the following objectives:

  1. Researched the differences between four Linux distributions and compared them with Fedora, with an emphasis on the Graphical User Interfaces (GUI)s
    • Ubuntu, OpenSUSE, Linux Mint, and Arch Linux were all the distributions compared with Fedora

  2. Fedora Package Reviews
    • Read the official Fedora User Documentation and Package Review Guidelines
    • Became Fedora package review contributers by reviewing packages on Red Hat's Bugzilla site which is the place where all new packages that are requested to be added to Fedora must be shared first to check for package development errors before being approved to be added to Fedora

  3. Packaging Open-Source Software Tools
    • Found free open-source software tools and projects from mainly Pypi and GitHub that are specifically aimed towards improving GUI appearance
    • Built the tools into Fedora packages following the package development guidelines
    • Submitted to Bugzilla for review and requested a necessary sponser for first time package maintainers

How to Install Fedora

There are a few different ways to get Linux running on your computer. A common way is to install VirtualBox, which is a software designed by Oracle to run Linux on a virtual platform within a local system.
This is a great way to try out Fedora and multiple other Linux distributions for the first time and is relatively easy to set up.

You can download VirtualBox from Oracle's VirtualBox website

    The official Fedora Website lists all the different types of Fedora downloads available.

      Select Fedora Workstation, which is a ready-to-go Fedora system that comes with many useful apps and software set up and ready to go.
      The download files are called ISO images or an .iso file, which stands for "International Organization for Standardization". The .iso files contain all the data needed to set up the Linux distribution.

      Download Fedora Workstation and select the ISO image download for the lastest version.

    Now you will need to set up your Fedora ISO image on VirtualBox. Follow this video tutorial by r2schools to complete the set-up process.

How to Install and Use Fedora-Review Tools

Fedora-Review is a tool that is used to review Fedora packages through the command line and to make sure they are compliant with Fedora's packaging guidelines. Here are the steps to install Fedora-Review using the DNF package manager:

    Open a terminal and enter the following command to install Fedora-Review:

    sudo dnf install fedora-review

    Verify that Fedora-Review was installed correctly by entering the following command:

    fedora-review --version

    We can then begin reviewing packages. On Bugzilla's site for new packages find a package that needs reviewing in the New tickets section. Each pacakge has a specific number attached to it. This is known as the bug or ticket number of a package.


    In the terminal, run this command with the package's ticket number:

    fedora-review -b 'bz_num'

    This will download the package and automatically review it, creating a text file containing a checklist of requirements that the package satisfies and any errors in the build attempt. Using this as a guide, Fedora package review contributers are able to create a reply message to the package's ticket with a review of what needs to be fixed.

What is an RPM Tree and How to Set it Up for Fedora Packaging Development

Packages that are built are created as .rpm files. RPM stands for Red Hat's Package Maintainer, and is used to build and maintain packages. Fedora packaging uses file categorization in a folder system to categorize the different files necessary to build a package. This is called an RPM tree, which conisits of five folders: BUILD, BUILDROOT, RPMS, SOURCES, SPECS, and SRPMS.
Here's how to set up an RPM tree for Fedora packaging development:

    Install the RPM development tools:

    sudo dnf install fedora-packager rpmdevtools

    Create an RPM tree or directory structure:

    rpmdev-setuptree

    The BUILD directory contains the files that are created during the build process. The RPMS directory contains the binary RPM packages that are built. The SOURCES directory contains the source files for the package. The SPECS directory contains the spec files that describe how to build the package, and the SRPMS directory contains the source RPM packages that are built.

    Next, we need to add the source files to the SOURCES folder. The sources file will usually either be a .tar.gz or a .zip file and can be downloaded from its online source and then moved to the SOURCES directory using this command:

    cp /path/to/source/file.tar.gz ~/rpmbuild/SOURCES/

    or the source can be added from the command line using an online link to the files with the wget command:

    wget 'package-link-here' -P ~/rpmbuild/SOURCES

Now the next step is to create a SPEC file.


What Are SPEC Files and How to Create Them?

SPEC files are text-based files that contain the instructions for RPM to build the package. This includes pacakge descriptions, source files, dependencies, and build instructions for creating the package.

To create a SPEC file for an open source package, run this command which will create a draft or outline of the SPEC file:

rpmdev-newspec ~/rpmbuild/SPECS/my-project.spec

Next, edit the SPEC file and fill in the information specfic to the package such as the name, version, build dependencies, and more.


This is the main development stage of package creation. For more information of writing SPEC files visit the SPEC files introduction at the Fedora Docs Website and their tutorial on Creating a Basic SPEC File

Once the SPEC file is finished, move on to the next step.


How to Build Packages from SPEC files

Use this command to build the package from the SPEC file using the RPM tool:

rpmbuild -ba ~/rpmbuild/SPECS/my-project.spec

If there are no build errors, you should see a message at the end that says "Complete!"


If there are any errors, it is most likely with the SPEC file. The command line should display a message explaining what the error is and on which line of the SPEC file it is located on. Fix these errors and run the command again.

How to Install Packages


Now that the RPM file has succesfully built, you can install the package using this command:

sudo dnf install -y ~/rpmbuild/RPMS/noarch/developed-package-name-here.noarch.rpm

You can find the RPM package name in the noarch folder located within the RPMS folder.


Congratulations! You have succesfully created an RPM package and installed it on your local system!

How to Submit Packages to Bugzilla for Review


Following the guidelines from Fedora Docs New Package Process for New Contributers , a Fedora package review contributer should be able to create a ticket for a package to be reviewed on Red Hat's Bugzilla website.
